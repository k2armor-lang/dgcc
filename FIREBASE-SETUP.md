# 교사 대시보드 설정 (Firebase · 실시간 취합)

모둠 태블릿이 각자 활동해도, 결과가 **교사 화면 한 곳에 실시간으로 모이게** 하려면 무료 Firebase를 한 번만 연결하면 됩니다. (약 10분, 구글 계정 필요)

> 설정을 안 해도 앱은 정상 작동합니다. 그 경우 취합은 결과 화면의 **"결과 코드 복사"** → 교사 대시보드 **"결과 코드 붙여넣기"** 로 수동 진행합니다.

---

## 1) Firebase 프로젝트 만들기
1. https://console.firebase.google.com 접속 → **프로젝트 추가** → 이름 입력(예: `digital-gatekeeper`) → 생성.
2. 왼쪽 **빌드 → Firestore Database** → **데이터베이스 만들기** → **테스트 모드로 시작** 선택 → 위치 아무거나 → 사용 설정.
   - (테스트 모드는 30일 후 잠깁니다. 아래 3)에서 규칙을 넣어 두면 계속 쓸 수 있습니다.)

## 2) 웹앱 등록하고 설정값 복사
1. 프로젝트 개요 옆 **⚙️ → 프로젝트 설정** → 아래로 스크롤 → **내 앱**에서 **웹(</>)** 아이콘 클릭.
2. 앱 닉네임 입력 → 등록. `firebaseConfig = { ... }` 값이 나옵니다.
3. 그 중괄호 안의 값을 복사해 **index.html** 의 아래 부분에 붙여넣으세요:

```js
const FIREBASE_CONFIG = {
  apiKey: "AIza............",
  authDomain: "your-app.firebaseapp.com",
  projectId: "your-app",
  appId: "1:xxxx:web:xxxx"
};
```
> apiKey는 비밀이 아닙니다(공개돼도 됨). 보안은 아래 3) 규칙으로 합니다.

### 세션 값 (기본: 모든 모둠 한 곳에 모음)
바로 아래 `DEFAULT_SESSION` 은 기본값 `"default"` 로 되어 있어, **반 구분 없이 모든 모둠이 한 곳에 모입니다.**
학생은 **모둠명·모둠원만** 입력하면 자동 제출되고, 교사 대시보드는 열자마자 자동 접속합니다.

```js
const DEFAULT_SESSION = "default";
```
> 새 수업을 이전 기록과 분리하고 싶을 때만 값을 바꾸세요(예: `"2교시"`). 이전 데이터는 Firebase 콘솔에서 지울 수 있습니다.

## 3) Firestore 보안 규칙 (권장)
Firestore → **규칙** 탭에 아래를 붙여넣고 **게시**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 수업용: 이 앱이 쓰는 경로만 읽기/쓰기 허용
    match /dgk_sessions/{session}/groups/{group} {
      allow read, write: if true;
    }
  }
}
```
- 누구나 읽고 쓸 수 있으니, **세션 코드**(예: `2026-3반-0718`)를 매 수업 새로 정해 사용하세요.
- 학생에게는 실명 대신 **모둠명**만 쓰게 하세요.

## 4) 사용 흐름 (완전 자동)
1. 학생: 인트로에서 **모둠명 + 모둠원**만 입력하고 활동.
2. 학생: 마지막 결과 화면에서 **제출 완료**가 자동 표시(누를 것 없음).
3. 교사: 상단 **교사용** → **자동 접속** → 모둠이 끝낼 때마다 **실시간**으로 쌓임.
4. **스테이지별 판단 근거 비교**를 펼쳐 다같이 보며 이야기.

## 참고
- 무료(Spark) 요금제로 한 학급 수업은 충분합니다.
- 오프라인 배포본(`offline-classroom/index-offline.html`)은 Firebase가 꺼진 채로 빌드되어, 취합은 결과 코드 방식만 씁니다.
